# AI PM 100 Projects OS

## 🎯 Objective

Build a portfolio that proves:

* 0→1 product execution
* Deep AI systems thinking (RAG, agents, evals)
* Strong product judgment (tradeoffs, UX, metrics)
* Ability to operate like a product owner in ambiguity

---

## 🧱 Portfolio Architecture

### Layer 1: Systems (25)

Infra + deep understanding
→ RAG, evals, agents, memory, observability

### Layer 2: Products (25)

Real-world AI SaaS-like builds
→ onboarding, support, analytics, personalization

### Layer 3: Experiments (20)

Original insights
→ benchmarks, tradeoffs, failure modes

### Layer 4: Teardowns (15)

Strategic thinking
→ product + system + business breakdowns

### Layer 5: Meta (15)

Authority layer
→ frameworks, playbooks, principles

---

## 🧠 Quality Bar (Non-negotiable)

Every project must answer:

1. Why does this problem matter?
2. Why is AI required (not optional)?
3. What system design choices were made?
4. What tradeoffs were consciously accepted?
5. What breaks at scale?
6. How would this evolve at 10x / 100x usage?

---

## ⚙️ Execution Modes

### 🟢 Shallow Mode (3–4 hrs)

Goal: Signal + clarity

Must include:

* Clear problem framing
* Working prototype OR strong conceptual design
* Thoughtful README

---

### 🔴 Deep Mode (3–5 days)

Goal: Career-defining assets

Must include:

* Full system design
* Real implementation
* Logging / evals / fallback logic
* Deployment (preferred)

---

## 📅 Weekly Operating Rhythm

Mon–Thu → Ship (1/day)
Fri–Sat → Deep build
Sun → Reflect + synthesize + improve past work

A portfolio that:

* Shows depth + breadth
